Event Bus — Step 2: Define Game Events & Trigger Mapping
========================================================
Branch: event-bus-zts
Date:   2026-03-30

Overview
--------
Created the typed event system that represents every meaningful thing that
can happen during a match. Also built the mapping table that connects the
existing TriggerType enum (from card effect data) to these new game events.
No existing files were modified — this is purely additive.

Behavior Changes
----------------
None. This file defines types, a mapping table, and helpers. No existing
code imports it yet. Zero runtime impact until Step 3 (event bus) wires
it in.

Files Added
-----------

lib/game-server/events.ts
  - GameEventType enum (13 event types):
      TURN_START, TURN_END, DAMAGE_DEALT, HEAL_APPLIED, BLOCK_APPLIED,
      STATUS_EFFECT_APPLIED, STATUS_EFFECT_EXPIRED, STATUS_EFFECT_TICK,
      CARD_PLAYED, CARD_EQUIPPED, CARD_DESTROYED, PLAYER_DIED, ENERGY_SPENT

  - 13 typed event payload interfaces, combined into a discriminated union
    (GameEvent). Directional events (DAMAGE_DEALT) carry sourcePlayerId
    and targetPlayerId. Non-directional events carry a single playerId.
    Most events carry sourceCard (MatchCard | null) to attribute the cause.

  - TRIGGER_MAP: Record<TriggerType, TriggerMapping[]>
    Maps each of the 12 TriggerType values to the GameEventType(s) that
    activate it, plus a perspective ("source", "target", "self", "any")
    that determines whose equipped cards are checked.
    Key mappings:
      ON_HIT         -> DAMAGE_DEALT (source perspective)
      ON_DAMAGE_TAKEN -> DAMAGE_DEALT (target perspective)
      ON_REBOUND     -> DAMAGE_DEALT (target perspective)
      START_OF_TURN  -> TURN_START (self)
      END_OF_TURN    -> TURN_END (self)
      ON_DEATH       -> PLAYER_DIED (self)
      ON_HEAL        -> HEAL_APPLIED (self)
      ON_BUFF        -> STATUS_EFFECT_APPLIED (target)
      ON_EQUIP       -> CARD_EQUIPPED (self)
      ON_USE         -> CARD_PLAYED (self)
      ON_SUMMON      -> CARD_PLAYED (self)
      PASSIVE        -> TURN_START (self)

  - getTriggersForEvent(event): given a GameEvent, returns all TriggerTypes
    that should fire and from which perspective. Used by the event bus to
    know which cards to check.

  - resolveTriggeredPlayer(event, perspective): given an event and a
    perspective, returns which player's cards to scan. Maps "source" to
    sourcePlayerId, "target" to targetPlayerId, "self" to playerId,
    "any" to "both".

Event Bus — Step 3: Build the Event Bus Dispatcher
===================================================
Branch: event-bus-zts
Date:   2026-03-30

Overview
--------
Built the in-process event dispatcher that sits between "something happened
in the match" and "card effects should fire." The bus takes a GameEvent and
the current MatchState, scans equipped cards for matching triggers, rolls
against trigger chances, and returns what fired. It does NOT mutate state —
it returns TriggeredEffect[] for the effect resolver (Step 5) to apply.

Behavior Changes
----------------
None. Purely additive — no existing code imports or calls event-bus.ts yet.
Will be wired into applyAction() in Step 4, and the resolveEffects callback
will be provided by the effect resolver in Step 5.

Files Added
-----------

lib/game-server/event-bus.ts

  Types:
  - TriggeredEffect: describes a card effect that fired (card, owner,
    triggerType, sourceEvent). Consumed by the effect resolver.
  - EmitResult: returned by emit() — list of triggered effects + event log.

  Card Scanning:
  - scanPlayerCards(player, playerId, triggerType, sourceEvent):
    iterates equippedItems (checks effectJson.trigger + triggerChance)
    and equippedTools (checks effectJson.conditionTrigger + conditionChance).
    Field names match the existing Zod schemas in card-effect-schemas.ts.
  - rollChance(chance): percentage roll (0-100). 100 = always, 0 = never,
    between = Math.random() * 100 < chance.

  Core Dispatch:
  - emit(event, state, depth?): main entry point.
    1. Checks depth < MAX_EVENT_CHAIN_DEPTH (10) — prevents infinite loops
    2. Checks match is active with both players
    3. Calls getTriggersForEvent() to find matching TriggerTypes
    4. Calls resolveTriggeredPlayer() to determine whose cards to scan
    5. Calls scanPlayerCards() for each player
    6. Returns TriggeredEffect[] and event log

  Cascade Loop:
  - emitWithCascade(event, state, resolveEffects, depth?): full chain handler.
    Accepts a resolveEffects callback (dependency injection for Step 5).
    1. Calls emit() to collect triggers
    2. Passes triggers to resolveEffects callback
    3. Callback returns updated state + any new GameEvents produced
    4. Recursively cascades new events at depth + 1
    5. Accumulates all effects/events across the chain
    6. Returns finalState, allEffects, eventLog

  Design Decisions:
  - Bus is pure (no state mutation) — separation from effect resolution
    keeps it testable and prevents ordering bugs
  - resolveEffects is a callback, not a direct import — avoids circular
    dependency between event-bus.ts and effect-resolver.ts (Step 5)
  - Depth counter is threaded through recursion, not stored as mutable
    state, so concurrent matches can't interfere

Event Bus — Step 4: Refactor applyAction() to Emit Events
==========================================================
Branch: event-bus-zts
Date:   2026-03-30

Overview
--------
Rewired applyAction() so every game action produces typed GameEvents that
flow through the event bus from Step 3. The state mutation arithmetic is
unchanged — same damage, same block, same heal — but now each mutation
emits an event and collects any card triggers that fire. A TODO marks
where Step 5 (effect resolver) will process those triggers.

Behavior Changes
----------------
For card-less matches: identical. emit() scans empty card arrays, finds
nothing, returns no triggers. All 45 tests pass unchanged.

For card-based matches: events now flow through the bus. Equipped items
with ON_HIT, ON_DAMAGE_TAKEN, START_OF_TURN, etc. will be detected and
returned as TriggeredEffects. Not yet applied (Step 5).

New: turnNumber increments by 1 on each turn swap (was static at 1).
New: HEAL caps at attacker.maxHp instead of GAME.MAX_HP (identical for
     default matches where maxHp === 30).

Files Changed
-------------

lib/game-server/match.ts
  New imports:
  - DamageType from enums (for DAMAGE_DEALT events)
  - GameEvent, GameEventType from events.ts
  - emit, TriggeredEffect from event-bus.ts

  applyAction() rewritten with this flow:
    1. Validation (unchanged)
    2. Emit TURN_START → collect triggers via emit()
    3. Apply action with same arithmetic → emit corresponding event:
       PUNCH/KICK → DAMAGE_DEALT (PHYSICAL, sourceCard: null)
       BLOCK     → BLOCK_APPLIED
       HEAL      → HEAL_APPLIED (capped at maxHp, not GAME.MAX_HP)
    4. Win check:
       - hp <= 0: emit PLAYER_DIED (before setting finished, so ON_DEATH
         triggers fire while match is still active), then set winner
       - alive: emit TURN_END, swap turn, increment turnNumber
    5. TODO: process collectedEffects through effect resolver (Step 5)
    6. Persist + publish (unchanged)

  Event ordering per action:
    TURN_START → [ACTION_EVENT] → TURN_END (or PLAYER_DIED)

  collectedEffects is populated but not yet consumed — the array is
  built and ready for the effect resolver callback in Step 5.

Branch: event-bus-zts

Overview

Built the module that takes TriggeredEffects from the event bus and turns

status effects, granting stat bonuses, and producing secondary events for
cascading. Wired it into applyAction() to complete the full pipeline.

Behavior Changes

Files Added
-----------

lib/game-server/effect-resolver.ts

  Target Resolution:
  - resolveTargets(targetType, ownerPlayerId): maps TargetType enum to
    actual PlayerIds. SELF/ALLY→owner, ENEMY→opponent, ALL→both.

  State Mutation Helpers:
  - applyDamage(): calculates block absorption, reduces HP, logs with
    card name attribution, produces DAMAGE_DEALT + PLAYER_DIED events.
  - applyHealing(): caps at maxHp, logs, produces HEAL_APPLIED event.
  - applyStatusEffect(): respects MAX_STATUS_EFFECTS cap (5), refreshes

    bonuses (healthBonus→maxHp+hp, defenseBonus→block). Moves consumable
    items to graveyard + emits CARD_DESTROYED.
  - resolveToolEffect(): applies damage, healing, status effects (simpler
    than items — no stat bonuses or consumable flag).

  Entry Point:
  - resolveTriggeredEffects(effects, state): routes each TriggeredEffect
    to item or tool resolver based on card.type. Returns updatedState +
    newEvents. This is the callback for emitWithCascade().

Event Bus — Step 7: Add New Action Types (PLAY_SPELL, USE_TOOL)
================================================================
Branch: event-bus-zts
Date:   2026-03-31

Design Decisions Made
---------------------
  D1. Energy regeneration: Full refill each turn
  D2. Spell costs: Rarity-scaled defaults (COMMON=1, UNCOMMON=1, RARE=2,
      EPIC=2, LEGENDARY=3), per-card manaCost overrides, minimum 1
  D3. Tool active use: Free, once per turn
  D4. Spell consumption: Moves to graveyard on use
  D5. Actions per turn: Multiple actions limited by energy. Spells and
      tools don't end the turn. Basic actions (PUNCH/KICK/BLOCK) end
      the turn. Players can cast spells and use tools before finishing
      with a basic action.

Overview
--------
Added PLAY_SPELL and USE_TOOL as new action types. Players can now play
spell cards from their hand (costs energy, spell goes to graveyard) and
actively use equipped tools (free, once per turn). Both happen within a
turn without ending it — the turn ends when the player uses a basic
action (PUNCH/KICK/BLOCK). Energy fully refills at the start of each
turn.

Behavior Changes
----------------
For card-less matches: identical. No spells in hand, no tools equipped,
basic actions work exactly as before. 43/43 tests pass unchanged.

For card-based matches: players can now cast multiple spells per turn
(limited by energy) and use one tool before finishing with a basic
action. Spell effects apply based on spellClass (DAMAGE, HEALING, BUFF,
DEBUFF, UTILITY, SUMMON, ENVIRONMENTAL). Tool active use always fires
(no chance roll — that's for passive triggers only).

Event Bus — Step 8: Status Effect Tick System
==============================================
Branch: event-bus-zts
Date:   2026-03-31

Design Decisions Made
---------------------
  D6. Freeze/Stun: probabilistic — 20% skip turn entirely, 20% restrict
      to basic actions only (no spells/tools), 60% block only. Rolled
      once per turn when the effect ticks.
  D7. Status effect stacking: refresh duration only (no damage stacking)
  D8. Tick timing: start of affected player's turn, after TURN_START
      event but before action selection
  D9. Burn vs Poison: BURN=3 tick damage, POISON=2 tick damage

Overview
--------
Added status effect tick processing at the start of each turn. BURN and
POISON deal damage, REGEN heals, SHIELD grants block, SEND_TO_GRAVEYARD
destroys a random equipped card, and FREEZE/STUN roll for turn restriction
severity. Durations decrement each tick and expired effects are removed.
Tick damage can kill players (checked immediately after ticks).

Behavior Changes
----------------
For card-less matches: identical. No status effects are ever applied,
so processStatusEffectTicks returns immediately. 43/43 tests pass.

For card-based matches: status effects now actively impact gameplay.
BURN/POISON can kill before the player acts. FREEZE/STUN create variable
restrictions (skip/basic-only/block-only) adding unpredictability.

Files Changed
-------------

lib/game-server/constants.ts
  - Added BURN_TICK_DAMAGE (3), POISON_TICK_DAMAGE (2),
    REGEN_TICK_HEALING (3), SHIELD_TICK_BLOCK (3)
  - Added FREEZE_SKIP_TURN_CHANCE (20), FREEZE_BASIC_ONLY_CHANCE (20)

lib/game-server/types.ts
  - PlayerState: added turnRestriction: "none" | "block_only" | "basic_only"

lib/game-server/effect-resolver.ts
  - New export: TurnRestriction type
  - New export: TickResult interface
  - New export: processStatusEffectTicks(state, playerId):
      BURN: deal BURN_TICK_DAMAGE, emit TICK + DAMAGE_DEALT + PLAYER_DIED
      POISON: deal POISON_TICK_DAMAGE, same event pattern
      REGEN: heal REGEN_TICK_HEALING (capped at maxHp), emit TICK + HEAL
      FREEZE/STUN: roll for restriction severity, log outcome
      SHIELD: add SHIELD_TICK_BLOCK, emit TICK + BLOCK_APPLIED
      SEND_TO_GRAVEYARD: destroy random equipped card, emit CARD_DESTROYED
    After ticking: decrement remainingTurns, remove expired effects,

lib/game-server/match.ts

  - If restriction is skip_turn: emit TURN_END, swap turn, return early
  - Stores restriction in attacker.turnRestriction for subsequent calls
  - Enforces restriction before action switch: block_only rejects non-BLOCK,

  - Resets turnRestriction to "none" on turn swap
